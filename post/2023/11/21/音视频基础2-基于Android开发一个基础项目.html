JNI(Java Native Interface)

  • 概念

    • 一种编程框架，允许运行于JVM的JAVA程序去调用本地代码(c/c++以及汇编语言的代码)

  • 使用场景

    • 应用程序需要一些平台相关特性的支持，而JAV层没有(如OpenSL ES)
    • 调用成熟的或者已经存在的、用c/c++语言编写的代码库(ffmpeg)
    • 应用程序的某些关键操作对运行速度有较高的要求，这部分逻辑可以用c或者汇编来写，再通过JNI向JAVA层提供访问接口

  • 步骤

    1. 编写一个java类，并在某个方法签名的修饰符中加上native修饰符
    2. 使用javac命令编译第一步的java类，使之成为一个class文件
    3. 使用javah命令将第二步的输出作为输入，生成JNI的头文件
    4. 将JNI头文件复制到项目下的jni目录，并且建立一个cpp的实现文件实现该jni头文件中的函数
    5. 编写Android.mk文件，加入第四步的本地代码，利用ndk-build生成动态链接库
    6. 在java类中加载第五步生成的动态链接库
    7. 在java类中调用该native方法

  4. 生成jni文件

  • 参考：jni示例
  • 提示：高版本java中javah被javac代替，直接用javac -h就行（我买的这书也太老啦！！！）
  • 流程
    1. 编写.java源代码，其中用native关键字标明需要本地实现的函数。
    2. 使用命令 javac -h <输出位置> <源代码路径> 编译出.h头文件。
    3. 编写.cpp源代码实现.h文件中声明的native函数。
    4. 编译.h头文件和.cpp源文件生成.dll动态库，并移动到.java源代码中指定的位置。
    5. 使用命令 javac <源代码路径> 将.java源代码编译成.class字节码。
    6. 使用命令 java <类名> 执行类名.class文件，调用.dll库文件得到输出。
